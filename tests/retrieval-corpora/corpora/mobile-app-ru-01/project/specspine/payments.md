# Платежи и билеты

Покупка создаёт локальную попытку с уникальным ключом идемпотентности. После
возврата из банковского приложения клиент не считает оплату завершённой по
самому redirect. Он запрашивает сервер и показывает билет только при статусе
`confirmed`. Статусы `pending` и сетевой тайм-аут оставляют попытку для
последующей сверки; `declined` завершает её без билета.

Внешняя ссылка может открыть экран ожидания, но её разбор принадлежит
[глубоким ссылкам](deep-links.md). Незавершённые попытки сохраняются в
[локальном хранилище](local-storage.md). Событие аналитики отправляется только
после серверного подтверждения.
