# Push-уведомления

Этот документ владеет регистрацией и заменой токена доставки.

После получения нового push-токена клиент сохраняет его как ожидающий и
отправляет серверу вместе с идентификатором установки. Старый токен остаётся
активным до подтверждения регистрации нового. После подтверждения новый токен
становится текущим, а прежний удаляется. Такая ротация не допускает окна без
доставки. Повтор той же пары установка–токен идемпотентен.

Нажатие уведомления преобразуется во внутренний маршрут по правилам
[глубоких ссылок](deep-links.md). Оперативную остановку рассылки описывает
[реагирование на инциденты](incident-playbook.md). Сессионные ключи принадлежат
[аутентификации](authentication.md).
