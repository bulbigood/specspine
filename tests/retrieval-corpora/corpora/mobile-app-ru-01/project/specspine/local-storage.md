# Локальное хранилище

Приложение использует одну зашифрованную базу на профиль. Таблицы остановок,
маршрутов, билетов, избранного и фоновых операций мигрируют до запуска
основного интерфейса. Чтение экранов идёт из локальных представлений, поэтому
медленная сеть не блокирует навигацию.

Очередь намерений содержит тип операции, полезную нагрузку, базовую серверную
ревизию, число попыток и ближайшее время повтора. Запись пользовательского
изменения и постановка намерения выполняются атомарно. Воркер арендует строку,
передаёт её серверу и записывает результат. Он не определяет, какая версия
данных должна победить: разрешение конфликтов и приоритет удаления принадлежат
[автономной синхронизации](offline-sync.md).

База также хранит метаданные [кэша изображений](image-cache.md) и незавершённые
[платежи](payments.md), но правила их жизненного цикла задают владельцы этих
областей. Токены доступа в базе не сохраняются; см.
[аутентификацию](authentication.md).
