# Аутентификация

Короткоживущий токен доступа находится только в памяти, а ключ обновления — в
системном защищённом хранилище. Один координатор объединяет одновременные
запросы на обновление сессии. Ответ 401 повторяется ровно один раз после
успешного обновления.

Выход удаляет ключи, пользовательскую базу и регистрацию устройства. Регистрация
для доставки уведомлений выполняется отдельно в
[push-уведомлениях](push-notifications.md). Push-токен не является учётной
сессией и не обновляет доступ к API.
