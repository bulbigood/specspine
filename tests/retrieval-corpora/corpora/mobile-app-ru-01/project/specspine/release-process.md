# Выпуск приложения

Релиз проходит внутреннее тестирование, закрытую группу и поэтапную публикацию
10–50–100 процентов. Повышение доли блокируется при росте аварийных завершений,
ошибок запуска или незавершённых платежей.

Миграции [локального хранилища](local-storage.md) проверяются на двух предыдущих
версиях. Новые возможности скрыты [флагами](feature-flags.md), пока минимальная
версия не достигнет требуемого уровня. Процесс выпуска не определяет бизнес-
правила синхронизации, оплаты или push-доставки.
