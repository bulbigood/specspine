# Навигация

Корневой координатор содержит вкладки карты, маршрутов, билетов и профиля.
Каждая вкладка сохраняет собственный стек. После нехватки памяти восстанавливается
только последний подтверждённый маршрут; модальные окна не восстанавливаются.

Внешний переход сначала проходит через [глубокие ссылки](deep-links.md), затем
координатор проверяет сессию и наличие требуемых данных. Экран билета читает
результат из [платежей](payments.md). Все кнопки и заголовки следуют
[требованиям доступности](accessibility.md).

Навигация не владеет разбором URI, подтверждением покупки, масштабированием
шрифта или доставкой push-уведомлений.
