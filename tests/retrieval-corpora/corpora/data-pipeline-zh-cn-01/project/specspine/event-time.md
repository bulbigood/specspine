# 事件时间与水位线

## 职责

本文拥有 event time、watermark（水位线）和迟到事件策略。事件时间来自经过校验的
`occurred_at`，不能用接收时间替代。每个输入分区独立推进水位线，算子水位线取活跃
分区的最小值。

允许迟到窗口为 15 分钟。水位线之后但仍在允许迟到范围内的事件触发修正结果；
超过范围的超迟事件写入 late-event 隔离流，不再静默修改已关闭窗口。空闲分区在
5 分钟无数据后标记 idle，避免阻塞全局进度。
